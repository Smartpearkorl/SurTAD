## 3 评价指标的定义

本项目主要从“异常检测（二分类）”与“异常分类识别（多分类）”两个维度进行性能评估，所有指标均在**视频帧级别（Frame-level）**进行统计。

### 3.1 异常检测评价指标

* **准确率 (Accuracy)**
  * 定义：在所有测试视频帧中，模型预测正确的帧数所占的比例。
  * 公式：
    $$Accuracy = \frac{TP + TN}{TP + TN + FP + FN}$$
  * *(其中 TP 为真正例，TN 为真负例，FP 为假正例，FN 为假负例)*

* **综合 F1 分数 (F1-Mean)**
  * 定义：由于交通视频中正常帧数量远大于异常帧，为避免类别极度不平衡带来的指标虚高，我们分别计算“正常帧”和“异常帧”的 F1 分数，并取两者的调和平均值。
  * 公式：
    $$F1_{mean} = \frac{2 \times F1_{normal} \times F1_{anomaly}}{F1_{normal} + F1_{anomaly}}$$

* **帧级 f-AUC (Area Under ROC Curve)**
  * 定义：受试者工作特征曲线（ROC）下的面积。它反映了在不同判定阈值下，模型将真实异常帧的得分排在正常帧之前的概率。
  * 公式：
    $$AUC = \int_{0}^{1} TPR \, d(FPR)$$
  * *(其中 TPR 为真正例率，FPR 为假正例率)*

### 3.2 异常分类识别评价指标

* **平均准确率均值 (mAP, Mean Average Precision)**
  * 定义：用于衡量模型在 13 种具体交通异常子类上的综合识别能力。首先计算每一个子类别的平均精度（AP），再对所有类别的 AP 求算术平均值。
  * 公式：
    $$mAP = \frac{1}{N_{class}} \sum_{i=1}^{N_{class}} AP_i$$
  * *(其中 $N_{class}$ 为异常类别总数，$AP_i$ 为第 i 个类别的平均精度)*

* **宏平均 AUC (Macro f-AUC)**
  * 定义：采用“一对多（One-vs-Rest）”策略，分别计算每个子类别的 AUC 值，随后取算术平均。该指标能对样本量较少的“长尾类别”给予公平的权重考量。
  * 公式：
    $$Macro\ f-AUC = \frac{1}{N_{class}} \sum_{i=1}^{N_{class}} AUC_i$$
  * *(其中 $N_{class}$ 为异常类别总数，$AUC_i$ 为第 i 个类别在 One-vs-Rest 策略下的 AUC 值)*

## 4 第三方测试流程与方法

为确保验收过程的客观性，整个测试分为“场景挑选”、“端到端评估”和“可视化核验”三个自动化模块。

### 4.1 测试场景挑选模块
* **测试流程**：测试人员启动“测试场景挑选客户端”，系统会自动加载测试数据集的元数据配置。
* **功能测试**：
  1. 界面以树形结构展示所有异常类别及其包含的测试视频。
  2. 测试人员可手动勾选感兴趣的视频，或在面板中输入抽样数量，系统将自动在各个子类下进行均匀的随机抽样。
* **测试结果**：工具将所选出的视频清单导出为一份独立的 JSON 配置文件，锁定本次验收的测试范围。

### 4.2 端到端推理与性能评估模块
* **测试流程**：在终端启动自动化测试流水线，输入上一步生成的 JSON 配置文件路径以及模型权重文件。
* **性能测试**：
  1. 系统在后台自动加载视频数据进行正向推理，输出每一帧的“异常得分”与“子类别预测概率”。
  2. 结合真实的标注文件（Ground Truth），程序自动计算全局的 AUC、F1-Mean 以及 mAP。
* **测试结果**：性能测试结果直接在终端输出。同时，系统会在输出目录下生成一份详细的文本报告，内部包含各项指标数值以及 13 分类的**混淆矩阵**，作为量化验收凭据。

### 4.3 结果可视化验证模块
* **测试流程**：性能评估完成后，流水线会自动触发渲染程序，对抽样选出的视频进行逐帧（或末帧）的图像结果绘制。
* **功能测试**：针对单张图像，系统会自动将“原始画面”、“预测类别与真实类别的对比标签”以及“异常得分走势图”拼接在一起。若子类别预测错误，文本标签将以红色高亮警示。
* **测试结果**：功能测试的输出结果保存在指定的 demo 目录下。生成的图像下方会清晰展示真实异常发生的时间区间（色块背景）与模型输出的得分曲线（折线）。测试专家可通过直观观察**折线峰值是否落入色块区间内**，来人工复核模型的实际表现。
